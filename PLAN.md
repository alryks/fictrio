**Общие Правила**
Рабочая папка: `/Users/srnm/Documents/BMSTU/year_3/term_2/CourseWork/src`.

Перед каждой задачей ты должен:
- работать в отдельной ветке по convention: `feat/...`, `chore/...`, `fix/...`;
- не смешивать несвязанные изменения;
- после этапа запускать релевантные проверки: `bun run typecheck`, `bun run lint`, при необходимости `bun run build`;
- коммитить в формате `type: short imperative summary`, коммитить буквально каждое изменение файла;
- фиксировать в README или docs только реально работающие команды.

Основной приоритет: приложение должно соответствовать уже спроектированной БД из курсовой, а не придумывать новую модель.

Важно: если возникает вопрос, нехватка информации или проблема, которая требует моего вмешательства, сразу об этом пиши и заканчивай ответ.
